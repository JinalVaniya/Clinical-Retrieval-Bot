# Vector Store

This directory contains the FAISS vector database generated from the medical PDF documents used by the Clinical Retrieval Assistant.

## Purpose

The vector store enables semantic search over medical documents by storing dense vector embeddings generated from document chunks.

Instead of searching for exact keywords, the system retrieves information based on semantic similarity between a user query and indexed document content.

---

## Contents

```text
vectorstore/
│
└── db_faiss/
    ├── index.faiss
    └── index.pkl
```

### index.faiss

Stores the FAISS vector index used for similarity search.

### index.pkl

Stores document metadata and mappings between vectors and source document chunks.

---

## How the Vector Store Is Created

The vector database is generated by:

```bash
python extract_vector_data.py
```

The process performs the following steps:

1. Load PDF documents from the `Data/` directory.
2. Extract text using PyMuPDFLoader.
3. Split documents into chunks using RecursiveCharacterTextSplitter.
4. Generate embeddings using:

```text
sentence-transformers/all-MiniLM-L6-v2
```

5. Store embeddings and metadata in FAISS.

## Retrieval Workflow

```text
User Query
     │
     ▼

     ▼

     │
     ▼

     │
     ▼
Groq LLM
     │
     ▼
Generated Response
```

```python
chunk_overlap = 50
retriever_k = 3

```

## Notes

* The vector store is automatically regenerated when `extract_vector_data.py` is executed.
* The contents of this directory are generated artifacts and can be recreated from source documents.
